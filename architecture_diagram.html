Key structural observations:

• plotting/ is a pure output sink — no plotting module imports from any other project module. Data flows in one direction only.

• models/wrappers.py is the central hub — the only file that imports from both cross_val and plotting.

• cross_val.py is stateless w.r.t. models — it only uses sklearn and delegates all output to plotting.

• streamlit_app.py is the widest importer — it reaches all four module packages directly, including bypassing run_loops to call wrappers.py functions directly for PCA.
